Chapter 6: Object-orientation with prototypes
  • Prototypes are the blueprint
  • Prototypes are a convenient way to define types of objects, but they are a feature of functions
  • Serve a similar purpose to that of classes in classical object-oriented languages
  • Prototypes predominant use is to produce a classical style of object oriented code and inheritance
  • Prototype property (property available in all functions) is really only useful when used with a constructor.
  • Object Literals
  • var myObj = {}; myObj.name = "fred"; myObj.myObjMethod = function() {return true;};
  • Using Prototypes
  • Constructor Functions & the new keyword
  • function Ninja() {}
    Ninja.prototype.swingSword = function() {return true;}
    var ninja1 = new Ninja();
    ninja1.swingSword(); // true
    // important: ninja references the prototype of the constructor it is not copied.
    // so change to prototype at the constructor level would - change the value at the instance
